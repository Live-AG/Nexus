Управление серверами

Обработка предназначена для администрирования кластера серверов "1С:Предприятия" посредством сервера администрирования кластера (ras). 

Для подключения к серверу "1С:Предприятия" требуется, чтобы сервер администрирования (ras) был запущен.

Подробнее о запуске сервера администрирования см здесь.

Примечание. Поддерживается соединение с несколькими серверами администрирования. С помощью разных серверов администрирования можно администрировать несколько серверов, а в том числе и один и тот же сервер "1С:Предприятия".

Основные действия (добавление, изменение, удаление элементов) выполняются в левой части формы, в которой отображается дерево центральных серверов. Не поддерживается возможность удаления группы элементов.

Важно! Создание администратора с пустым именем запрещено.

На форме редактирования элемента присутствует ссылка Документация, при нажатии которой откроется соответствующая страница документации (для русского языка).
С помощью команд меню Ещё можно сохранить (Сохранить настройки в файл) или загрузить из файла (Загрузить настройки из файла) следующие настройки:

  • список центральных серверов,
  • пароли администраторов центральных серверов,
  • пароли администраторов кластеров и информационных баз.

При закрытии обработки список соединений сохраняется в хранилище общих настроек. При открытии обработки список соединений загружается из хранилища.

Отличия от консоли кластера серверов "1С:Предприятия"

В таблицах, отображающих элементы, нельзя редактировать элементы. Форма редактирования элемента открывается при двойном щелчке на элементе в дереве центральных серверов.

Для изменения значения числовых полей можно использовать кнопки регулирования.

Даты можно изменять с помощью встроенного календаря.

Рабочие сервера

Требования назначения отображаются при выборе рабочего сервера мышью.
Нет колонки IP порт.
Рабочие процессы по серверам не отображаются.
Менеджеры кластера по серверам не отображаются.

Список сеансов и список счетчиков потребления ресурсов

Свойства сеанса отображаются только в списке сеансов.

Для списка сеансов и списка значений счетчиков потребления ресурсов отображаемые результаты отображаются постранично, поддерживается поиск выбранного значения (команда контекстного меню Искать везде).

Также в данных списках можно устанавливать отбор по численным показателям.

Редактирование параметров рабочего сервера

В форме расположено табличное поле Диапазон IP портов для ввода и редактирования диапазона используемых портов.

Ограничения потребления ресурсов

В списке в колонке Счетчик отображается имя счетчика.

Соединения

В списке соединений отображается колонка Идентификатор соединения.

Блокировки

В списке блокировок нет колонок Инф. база, Соединение, Сеанс и Приложения.
Блокировки по сеансам не отображаются.

Рабочие процессы

В списке рабочих процессов нет колонок Активен и Включен. Значения данных параметров всегда Истина.
