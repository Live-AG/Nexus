Параметры рабочего сервера

Для просмотра и изменения свойств сервера кластера следует выбрать требуемый сервер в списке серверов кластера и выполнить команду контекстного меню Изменить или аналогичную команду в меню обработки.

Рабочий сервер описывается следующими параметрами:

Описание сервера, компьютер и номер IP-порта, установленные раннее отредактировать нельзя.

Общие параметры

  • Компьютер - Сетевой адрес рабочего сервера, на котором запущен агент сервера. ЗАМЕЧАНИЕ. Если в качестве сетевого адреса рабочего сервера "1С:Предприятия" (свойство Компьютер) указан IP-адрес в точечной нотации, то его присутствие в DNS (файл hosts) не требуется.
  • Описание - Произвольное описание сервера кластера.
  • Центральный сервер - Если флажок установлен, то с этим рабочим сервером будет выполняться синхронизация реестра кластера и адрес этого рабочего сервера может быть использован для подключения к кластеру клиентским приложением.
  • Менеджер под каждый сервис - Управляет выделение отдельного менеджера кластера под каждый тип сервиса. Если флажок установлен, то для каждого типа сервиса кластера будет создаваться свой менеджер кластера. В противном случае все сервисы кластера назначаются на один менеджер кластера данного рабочего сервера.
    СОВЕТ. Устанавливать данное свойство можно в процессе опытной эксплуатации.
  • IP Порт - Номер сетевого порта агента сервера, который запущен на указанном компьютере. По умолчанию - 1540.
  • Порт главного менеджера кластера - Номер сетевого порта главного менеджера кластера, запущенного на данном рабочем сервере. Этот сетевой порт используется при формировании адреса кластера серверов для указания клиентскому приложению. Адрес выглядит следующим образом: <Свойство Компьютер>:<Порт главного менеджера кластера>. Если свойство Компьютер имеет имя COMP1, а свойство Порт главного менеджера кластера равно 2541, то адрес кластера серверов будет выглядеть как COMP1:2541.
    Значение данного параметра игнорируется в том случае, если не установлен флажок Центральный сервер.
  • Диапазоны IP портов - Диапазон сетевых портов, который будет использоваться системой для назначения адресов рабочим процессам, создаваемым на этом сервере. По умолчанию - 1560:1591

Использование памяти

  • Безопасный расход памяти за один вызов - Объем памяти в байтах, использование которого в процессе вызова сервера считается безопасным.
    ПРИМЕЧАНИЕ. Доступно только для лицензии КОРП.
    Может принимать значение от -1 до 9 223 372 036 854 775 807:
    • -1 - любой вызов сервера считается опасным, если за время вызова сервера достигнут максимальный объем памяти рабочего процесса;
    • 0 - значение объема определяется автоматически, как 5% максимального объема памяти рабочих процессов на данном рабочем сервере.
  • Критический объем памяти процессов - размер критического объема памяти процессов кластера (рабочих процессов и менеджеров кластера). Суммарный объем виртуального адресного пространства, занимаемого процессами кластера на рабочем сервере, превышение которого может привести к значительному снижению производительности рабочего сервера.
    ПРИМЕЧАНИЕ. Доступно только для лицензии КОРП.
  • Временно допустимый объем памяти процессов -
    ПРИМЕЧАНИЕ. Доступно только для лицензии КОРП.
    Определяет размер временно допустимого объема оперативной памяти, занятой рабочими процессами и менеджерами кластера (процессами кластера)на настраиваемом рабочем сервере, в байтах. Если объем памяти, потребляемой процессами кластера, превышает значение параметра, то система считает, что у данного рабочего сервера снижается производительность и на него больше не следует назначать новые соединения с информационными базами.
    Может принимать значение от -1 до 9 223 372 036 854 775 807:
    • -1 - временно допустимый объем памяти, доступный процессам кластера на данном рабочем сервере, не ограничен;
    • 0 - временно допустимый объем памяти процессов кластера на данном рабочем сервере определяется автоматически как 80% объема оперативной памяти сервера.
  • Интервал превышения допустимого объема памяти процессов -
    ПРИМЕЧАНИЕ. Доступно только для лицензии КОРП.
    Определяет, на какое время объем оперативной памяти, потребленной процессами кластера, может превысить значение параметра Временно допустимый объем памяти процессов. Описание алгоритма приводится в описании параметра рабочего сервера Временно допустимый объем памяти процессов.
    Параметр будет использоваться только в том случае, если его значение не равно 0 и значение параметра Временно допустимый объем памяти процессов отлично от значения -1.
    Значение по умолчанию - 300 секунд.

Параметры рабочих процессов

  • Количество ИБ на процесс
    ПРИМЕЧАНИЕ. Доступно только для лицензии КОРП.
    Количество информационных баз, соединения с которыми могут обслуживаться одним рабочим процессом данного сервера. Значение 0 означает, что ограничение не установлено.
    Если количество информационных баз превысит это количество - кластер серверов создаст на этом рабочем сервере дополнительный рабочий процесс.
  • Количество соединений на процесс - Количество соединений с информационными базами, которые может обслуживать один рабочий процесс данного сервера. Значение 0 означает, что ограничение не установлено. Если количество соединений, обслуживаемых рабочим процессом, превысит это количество - кластер серверов создаст на этом рабочем сервере дополнительный рабочий процесс.
